# Правила управления проектом (Governance)

- Без команд владельца проекта не выполняются никакие изменения в GitHub (push, merge, release, tag).
- Имена веток, коммитов, файлов — на английском.
- Перед изменениями — краткий план и минимальный объём правок.
- Релизы проходят через тегирование (`vX.Y.Z`) и запись в CHANGELOG.
